项目名称：音乐推荐播放器

功能需求：

主页（推荐歌单、热门歌曲、个性推荐）

用户注册（邮箱、用户名、密码、确认密码）（做最简单实现）

用户登录（邮箱、密码）（做最简单实现）

个人信息管理（修改昵称、密码、头像等）

音乐推荐（显示歌曲热度/平台评分，以及用户个人喜好评分/标记）

评论（对歌曲或歌单发表评论）

支持歌词显示

关于页面

音乐列表显示（分页、搜索、按风格/歌手筛选）

音乐详情页（显示歌词、专辑信息、推荐相似歌曲）

在线播放（简单的播放控件：播放/暂停、上一曲/下一曲、进度条）

获取音乐数据流程：

调用爬虫从公开音乐 API 或站点（如网易云音乐,QQ音乐,咪咕音乐,酷狗音乐）获取热门歌曲、新歌榜、推荐歌单。

从移动端或 Web 端接口爬取歌曲元数据（歌名、歌手、专辑、封面图片、播放链接、平台热度/评分）。

将爬取的数据存入本地数据库，用于显示音乐列表及详情页。

每日0点更新音乐数据（如排行榜、新歌），保证推荐内容新鲜。

更新后将歌曲封面图片缓存到本地服务器，方便页面快速加载。

每次服务器启动时，检查是否需要更新（无新信息则不更新）。

技术栈：

页面显示：HTML + CSS + JavaScript（前端播放器用原生 Audio API）

服务器：Python + Flask（提供 RESTful API 及静态页面）

数据库：SQLite（存储用户、歌曲信息、评论、个人评分/喜好）

关键规范（必须遵守）：

错误处理：后端统一返回 { "code": "xxx", "message": "中文提示" }，前端根据 message 显示友好错误信息。

依赖管理：使用 pip 清华源（-i https://pypi.tuna.tsinghua.edu.cn/simple），锁定依赖版本（生成 requirements.txt）。

编码规范：所有文件使用 UTF-8 编码，避免 GBK 导致的乱码问题。

播放链接处理：若爬取的链接为临时或需防盗链，需在服务端做代理转发或添加 Referer 头。

个人评分/喜好：允许用户对歌曲标记“喜欢/不喜欢”，作为后续推荐依据（协同过滤简化版或基于标签）。

定时任务：使用 Flask-APScheduler 或系统 crontab 调用脚本，每日凌晨0点更新数据
